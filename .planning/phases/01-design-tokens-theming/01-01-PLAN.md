**File 1: `src/app/layout.tsx`** — Three changes:

1. Replace the Inter font import and definition with Newsreader + Roboto:
```typescript
import { Newsreader, Roboto } from "next/font/google";

const newsreader = Newsreader({
  subsets: ["latin"],
  weight: ["400", "500", "600", "700"],
  variable: "--font-serif",
  display: "swap",
});

const roboto = Roboto({
  subsets: ["latin"],
  weight: ["300", "400", "500", "700"],
  variable: "--font-sans",
  display: "swap",
});
```
Remove the `const inter = Inter({ subsets: ['latin'] })` line entirely.

2. Update the `<body>` tag className from `{inter.className}` to:
```tsx
<body className={`${roboto.variable} ${newsreader.variable} font-sans`}>
```
This sets Roboto as default body font (font-sans) and makes both CSS variables available for Tailwind.

3. Change ThemeProvider `defaultTheme` from `"system"` to `"light"` per user decision that light mode is default for new visitors:
```tsx
<ThemeProvider
  attribute="class"
  defaultTheme="light"
  enableSystem
  disableTransitionOnChange
  storageKey="transfers-daily-theme"
>
```

4. Update the inline theme-flicker script to default to light (not system dark detection) when no stored theme exists. Replace the condition `(!theme && window.matchMedia('(prefers-color-scheme: dark)').matches)` with just `(theme === 'dark')`:
```javascript
var theme = localStorage.getItem('transfers-daily-theme');
if (theme === 'dark') {
  document.documentElement.classList.add('dark');
} else {
  document.documentElement.classList.remove('dark');
}
```
