  <name>Task 2: Content distribution and article views API routes</name>
  <files>src/app/api/admin/content/distribution/route.ts, src/app/api/admin/content/article-views/route.ts, src/app/api/admin/content/translations/route.ts</files>
  <action>
Create three API routes following the Phase 2 pattern (validateAuth, unstable_cache, createClient from supabase/server).

**Route 1: /api/admin/content/distribution (GET)**
- `unstable_cache` with key `['content-distribution']`, revalidate: 300 (5 min, same as dashboard)
- Three parallel Supabase queries inside the cached function:
  a. League distribution: `from('articles').select('league').eq('status', 'published')` — then count in JS grouping by league. Map league names to colors using LEAGUE_BY_NAME from constants.ts (fallback color '#6b7280' for unknown leagues). Sort descending by count.
  b. Category distribution: `from('articles').select('category').eq('status', 'published')` — group by category in JS. Capitalize first letter. Sort descending by count.
  c. Articles per day (last 30 days): `from('articles').select('status, created_at, published_at').gte('created_at', thirtyDaysAgo)` — group by DATE of created_at. For each day, count published (status='published') and drafts (status='draft'). Gap-fill missing days with 0. Format label as "Mar 1" (same as Phase 2 traffic chart).
- Also compute totalPublished and totalDrafts from the articles-per-day data.
- Return ContentDistributionResponse.

**Route 2: /api/admin/content/article-views (GET)**
- Accepts `?slugs=slug1,slug2,slug3` query param (comma-separated, max 100).
- If no slugs provided, return empty `{ views: {}, cachedAt: ... }`.
- Check GA4 credentials (same pattern as traffic route — return `{ success: false, error: 'ga4_not_configured' }` if missing).
- `unstable_cache` with key `['content-article-views']`, revalidate: 3600 (1 hour, per D-08).
- GA4 `runReport` with:
  - pagePath dimension
  - screenPageViews metric
  - dimensionFilter: pagePath CONTAINS '/article/' (same as top-articles route)
  - dateRanges: last 30 days
  - limit: 500
- Extract slugs from paths using `path.split('/article/')[1]?.split('?')[0]` (same as top-articles).
- Aggregate across locales (same slug from /en/article/X and /fr/article/X).
- Filter to only requested slugs.
- Return `ArticleViewsResponse` with `views` as `Record<string, number>`.
- Per D-09: articles not in GA4 data won't appear in the map — UI shows dash.

**Route 3: /api/admin/content/translations (GET)**
- `unstable_cache` with key `['content-translations']`, revalidate: 300 (5 min).
- Two queries:
  a. Total published articles count: `from('articles').select('uuid', { count: 'exact', head: true }).eq('status', 'published')`
  b. Translations per language: `from('translations').select('language')` — group by language in JS, count per language.
- For each of the 5 SUPPORTED_LANGUAGES, compute:
  - `articleCount`: count of translations for that language (from query b)
  - For 'en': use totalPublished as articleCount (English is the original language, no translation needed)
  - `translatedCount`: same as articleCount for non-en; for en: totalPublished
  - `coveragePercent`: `Math.round((translatedCount / totalPublished) * 100)` (0 if totalPublished is 0)
- Language labels: `{ en: 'English', es: 'Spanish', fr: 'French', de: 'German', it: 'Italian' }`.
- Return `TranslationCoverageResponse`.

All three routes: import validateAuth from '@/lib/supabase/auth-guard', return 401 if not authenticated.
